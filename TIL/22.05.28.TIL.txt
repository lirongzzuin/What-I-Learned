Spring 숙련 주차 2일째 되는 날이다. 로그인 기능에 대한 강의를 봤는데 어려운 느낌이다.
스프링에서 제공하는 로그인 회원가입 기능을 사용하는 방법도 봤고, 카카오톡 로그인 기능을
활용한 OAuth 기능도 확인했고, 그리고 마지막으로 자세히는 아니지만 JWT 로그인 기능도
볼 수 있었다. 각각 차이점이 있었는데, 아무래도 카카오톡을 활용한 OAuth기능이 제일
낫지 않나싶다 DB관리 할 때 조금 어려워지는 부분이 있을 수 있지만, 그래도 사용 편의성에서는
제일 좋을 것 같다는 생각이다.
그런데 기능이 늘어날수록 확실히 테스트를 하는게 힘들어지는 느끼이었다. 3주차 강의에서는
테스트 코딩을 어떻게 하는지에 대해 배운다고 하니까 앞으로 많은 기능을 다루면서 테스트를 해야하는
상황에 유용하게 활용할 수 있을 것 같다.
인증과 인가의 차이점을 알게 되었다.
Authorization / Authentication
로그인 과정에서 사용되는게 Authentication '인가'이고,
정보에 접근할 수 있는 권한이 주어지는 기능이 Authorization '인증'이었다.
DB를 관리하고 서버를 운영하는 백엔드에게 이 두 가지는 참 중요한 개념일 것 같기에 더 조사하고
공부해야 할 것 같다.
