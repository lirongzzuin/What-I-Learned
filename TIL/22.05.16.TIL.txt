알고리즘 문제를 28번까지는 최소 풀어야 하는데 지금 19번까지 왔다. 알고리즘 문제를 다 풀었다기 보다는 풀려고 노력을 해보고
안 풀리겠다 싶으면 구글링해서 답안지를 보고 왜 그렇게 풀었는지 풀이를 듣고 이해하는 방식으로 공부했다.
안 풀리는 문제를 끝까지 붙들고 푸는 것도 좋은 방법일 수 있겠지만, 자바를 처음 접해보고 어떻게 동작하는지 아직까지는 이해하기
힘들기 때문에, 차라리 빠르게 답지를 보고 왜 이렇게 풀었는지 해석을 읽어보고 내 스스로 생각하는 방식을 택했다.
20개 가까운 알고리즘 풀이들을 보고 내 스스로 풀려고 노력도 해보고 히니까, 첫 날에 느꼈던 막막한 기분은 어느정도 사라진 것 같다.
이제 몇몇 문제들은 아~ 이런식으로 풀면 되겠구나 감이 오고, (문제는 그걸 코딩으로 풀어서 쓰질 못한다는게 문제 ㅋㅋㅋ)
내 스스로 보자마자 푼 문제도 있다는 점에서 많은 발전이 있었으리라 짐작해본다. 그래도 아직은 부족하고 많은 노력들이 필요하기 때문에
앞으로 남은 알고리즘 기간동안에도 계속해서 JAVA와 친해지려 노력해야 될 것 같다.
일단 20번 문제에는 HashMap이라는 개념이 나오는데 앞서 지금까지는 거의 다부분 String, Integer 배열을 사용해서 문제를 풀었는데,
이번에는 배열의 길이가 다르고 다른 배열을 비교해야 하는 문제였기 때문에 HashMap을 사용했다.
HashMap은 순서는 딱히 없고 key, value 값을 가진 2차원 배열?이라고 생각 하면 편할 것 같다.
예를 들어 대학교에서 학생이름과 학번을 같이두고 찾는 느낌?? 배열에서는 뭔갈 더할때 .add를 사용한다면 HashMap에서는 .put을 사용하고
getOrDefault -> 해당하는 값이 있으면 가져오고 아니면 Default 0으로 해라.
get(key) -> key에 해당하는 value를 가져와라.
등의 새로운것들을 배웠다. 이런것들이 모두 익숙해져야 자바 쓰기 더 용이할듯 하다.
